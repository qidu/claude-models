# How to Export Chosen ENV Variables to Your Shell

## The Problem
When the module sets `process.env.VAR = value` in Node.js, it **only affects the current Node.js process**. Your shell doesn't see these variables.

## Solutions

### Option 1: Use the Export-Focused Script (Easiest)
```bash
cd claude-models
node export-focused.js claude-models.txt
```

This generates:
1. `claude-env.sh` - Shell script with exports
2. `.claude.env` - Dotenv file
3. Shows one-line export command

**To actually export:**
```bash
# Method A: Source the script
source claude-env.sh

# Method B: Copy-paste the one-line export
export ANTHROPIC_BASE_URL="https://api.qnaigc.com" ANTHROPIC_AUTH_TOKEN="sk-..." ANTHROPIC_MODEL="minimax/minimax-m2.5" API_TIMEOUT_MS="600000"

# Method C: Use dotenv file
export $(cat .claude.env | xargs)
```

### Option 2: Use the Enhanced TUI
```bash
cd claude-models
node final-enhanced.js
```

After selection, it will:
1. Generate `claude-export.sh`
2. Show export commands
3. Tell you how to source

### Option 3: Manual Export
After running the TUI and seeing the selected values:

```bash
# Copy these exact lines into your terminal
export ANTHROPIC_BASE_URL="https://api.qnaigc.com"
export ANTHROPIC_AUTH_TOKEN="sk-2dd63001b32cf7d9***"
export ANTHROPIC_MODEL="minimax/minimax-m2.5"
export API_TIMEOUT_MS="600000"
```

## Verification
After exporting, verify with:
```bash
echo $ANTHROPIC_BASE_URL
echo $ANTHROPIC_MODEL
echo $API_TIMEOUT_MS
```

## Using Claude
Once exported, you can run:
```bash
claude
```

The `claude` command will now see the environment variables.

## Important Notes

1. **Shell session only** - Exported variables last only for your current terminal session
2. **Permanent setup** - For permanent setup, add exports to your shell profile:
   ```bash
   # Add to ~/.bashrc, ~/.zshrc, or ~/.profile
   export ANTHROPIC_BASE_URL="https://api.qnaigc.com"
   export ANTHROPIC_AUTH_TOKEN="sk-..."
   export ANTHROPIC_MODEL="minimax/minimax-m2.5"
   export API_TIMEOUT_MS="600000"
   ```
3. **Security** - API tokens in shell history; consider using `.env` files or secret managers

## Quick Start
```bash
# 1. Go to module directory
cd claude-models

# 2. Generate exports
node export-focused.js

# 3. Source the generated script
source claude-env.sh

# 4. Verify
echo $ANTHROPIC_BASE_URL

# 5. Use claude
claude
```

## Generated Files
- `claude-env.sh` - Shell script with export commands
- `.claude.env` - Dotenv format file
- `claude-export.sh` - Generated by enhanced TUI

Choose the method that works best for your workflow! 🐾
